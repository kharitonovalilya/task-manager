## Цели

- Создать инструмент для командной работы над задачами
- Реализовать систему аутентификации и авторизации на основе JWT
- Разграничить права: лид команды может управлять участниками и задачами, участник — только видеть и отмечать выполнение своих задач
- Предоставить календарный вид для отслеживания дедлайнов
- Упаковать всё в Docker‑контейнеры

## Роли и права доступа

![roles and permissions](Diagrams/roles%20and%20permissions.png)

### Участник команды
- Просмотр списка своих команд.
- Просмотр участников и задач команды.
- Переключение статуса **только своих задач**.

### Лид команды
- Обладает всеми правами участника.
- Создание и удаление команды.
- Добавление и удаление участников.
- Создание, редактирование, удаление **любых задач** в своей команде.
- Переключение статуса **любой задачи** команды.

## Инструкция к запуску 

1. Откройте Docker Desktop
2. Запустите файл ```docker-compose.yaml``` командой ```docker-compose up --build```, находясь в директории с этим файлом
3. Сервесы доступны по адресам: 
 - ```localhost/register.html``` - регистрация
 - ```localhost:5432``` - PostreSQL база данных
 - ```localhost:80``` - фронтенд
 - ```localhost:8000``` - бэкенд
