## Цели

- Создать инструмент для командной работы над задачами
- Реализовать систему аутентификации и авторизации на основе JWT
- Разграничить права: лид команды может управлять участниками и задачами, участник — только видеть и отмечать выполнение своих задач
- Предоставить календарный вид для отслеживания дедлайнов
- Упаковать всё в Docker‑контейнеры

## Роли и права доступа

![roles and permissions](Diagrams/roles%20and%20permissions.png)

### Участник команды
- Просмотр списка своих команд.
- Просмотр участников и задач команды.
- Переключение статуса **только своих задач**.

### Лид команды
- Обладает всеми правами участника.
- Создание и удаление команды.
- Добавление и удаление участников.
- Создание, редактирование, удаление **любых задач** в своей команде.
- Переключение статуса **любой задачи** команды.


## Взаимодействие компонентов

### Диаграмма взаимодействия компонентов при входе
![login](Diagrams/login%20sequence%20diagram.png)

1. Пользователь отправляет `POST /auth/login` с email и паролем.
2. Бэкенд находит пользователя в базе данных.
3. Сервис безопасности проверяет пароль (bcrypt).
4. При успехе генерируется JWT‑токен и возвращается клиенту.
5. Клиент сохраняет токен и использует его в заголовке `Authorization: Bearer`.

### Диаграмма взаимодействия компонентов для создания задачи
![create_task](Diagrams/create_task%20sequence%20diagram.png)

1. Лид отправляет `POST /tasks` с данными задачи.
2. Бэкенд проверяет, является ли пользователь лидом указанной команды.
3. Дополнительно проверяется, что назначенный исполнитель состоит в этой команде.
4. При успешной проверке задача сохраняется в базе данных и возвращается клиенту.


## Инструкция к запуску 

1. Откройте Docker Desktop
2. Запустите файл ```docker-compose.yaml``` командой ```docker-compose up --build```
3. Сервесы доступны по адресам: 
 - ```localhost/register.html``` - регистрация
 - ```localhost:5432``` - PostreSQL база данных
 - ```localhost:80``` - фронтенд
 - ```localhost:8000``` - бэкенд
